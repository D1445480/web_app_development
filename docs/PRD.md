# 個人記帳簿系統 PRD (Product Requirements Document)

## 1. 專案概述
- **背景與動機**  
  現代人生活開銷繁雜，若無記帳習慣，常不知金錢流向，導致超支或無法達成理財目標。這份專案旨在打造一個簡單、清楚、好用的記帳工具，幫助人們輕鬆管理自身的日常財務。
- **目標用戶**  
  一般大眾皆可使用（對財務管理有需求，且需要輕量化工具的個人）。
- **核心價值主張**  
  一個幫助使用者快速記錄與分析收支的個人記帳系統，讓你掌控每分錢的流向。

## 2. 功能需求

### 主要功能列表與使用者故事
1. **收支快速記錄**
   - *使用者故事*：作為一般用戶，我希望能夠快速記錄每一筆收入與支出，以便隨時追蹤每日的金錢流動。
2. **類別分類統計**
   - *使用者故事*：作為一般用戶，我希望能夠為每一筆收支給予特定的類別（如餐飲、交通、娛樂），以便了解各類別的開銷比例與消費習慣。
3. **月度收支總覽**
   - *使用者故事*：作為一般用戶，我希望能在一個專屬頁面上查看每個月的總收入、總支出與結餘，以便對當月的整體財務狀況一目瞭然。
4. **搜尋＆篩選功能**
   - *使用者故事*：作為一般用戶，我希望可以根據日期、類別、金額或關鍵字搜尋過去的記錄，以便快速找到特定的歷史帳目。
5. **預算提醒**
   - *使用者故事*：作為一般用戶，我希望能夠設定每月的開銷預算上限，並在花費快要超支時獲得系統的視覺提醒，以便即時控制未來開銷。

## 3. 非功能需求
- **技術限制**：
  - 後端將採用 Python Flask 框架。
  - 前端渲染使用 Jinja2 標籤。
  - 資料庫使用輕量化的 SQLite，方便個人佈署與使用。
- **效能與安全考量**：
  - 頁面載入速度需快速（如：1秒內呈現當月帳目），給予流暢的使用體驗。
  - 確保資料表操作使用 ORM 或參數化查詢，防止 SQL Injection 安全風險。
  - （若是單人本地端系統則略過；若為連線版本需考量基本的使用者 Session 管理機制）。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have（必須具備）**
  - 新增、修改、刪除、檢視每日收支（基本的 CRUD）。
  - 可以將帳目歸入預設好的類別。
  - 簡易純文字的月度收支表/清單。
- **Should Have（應該具備）**
  - 圖表呈現各類別比例（圓餅圖或長條圖等類別分類統計）。
  - 基本的紀錄搜尋與篩選（依日期區間、類別）。
- **Nice to Have（最好具備，後續擴充）**
  - 月度預算設定與超支提醒的儀表板。
  - 可自訂收支類別（新增/編輯/刪除）。
  - 將資料匯出成 CSV 檔案的功能。
