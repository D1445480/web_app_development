# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述
- **背景與動機**：現代人生活花費繁雜，往往到了月底不知道錢花在哪裡。為了解決這個問題，我們需要一個簡單且直觀的記帳工具來有效管理個人財務。系統描述為：一個幫助使用者快速記錄與分析收支的個人記帳系統。
- **目標用戶**：一般大眾（想要輕鬆記錄並掌握自己收支狀況的人）。
- **核心價值主張**：提供一個能快速記錄與清晰分析收支的系統，幫助使用者輕鬆管理金錢。

## 2. 功能需求
系統共涵蓋以下五大主要功能，各對應其使用者故事：

1. **收支快速記錄**
   - 作為 **一般用戶**，我希望 **能以最少的操作步驟快速新增一筆收入或支出紀錄**，以便 **隨時隨地記帳而不感到麻煩**。
2. **類別分類統計**
   - 作為 **一般用戶**，我希望 **能查看不同類別（如餐飲、交通、娛樂）的收支統計**，以便 **了解自己主要的消費去向**。
3. **月度收支總覽**
   - 作為 **一般用戶**，我希望 **能看到每個月的總收入、總支出與結餘圖表**，以便 **掌握當月的財務健康狀況**。
4. **搜尋＆篩選功能**
   - 作為 **一般用戶**，我希望 **能根據日期、類別或關鍵字搜尋過去的記帳紀錄**，以便 **快速對帳或查詢特定花費**。
5. **預算提醒**
   - 作為 **一般用戶**，我希望 **能設定各類別或整體的月度預算，並在快超支時收到提示**，以便 **控制開銷並達成財務目標**。

## 3. 非功能需求
- **技術限制**：系統預設將使用 Flask 框架搭配 Jinja2 模板引擎進行前端頁面渲染，並使用 SQLite 作為輕量級的資料庫儲存方案。
- **效能考量**：保持簡單輕量的系統架構，確保頁面操作流暢度，提供極速的記帳體驗。
- **安全考量**：防範常見的網頁漏洞（如 SQL Injection），確保基本的資料讀寫安全。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 收支快速記錄（新增、更新、刪除）
  - 基礎類別分類設定（提供幾種預設的民生類別）
  - 條列式顯示與月度收支總覽
- **Should Have (應該有)**：
  - 類別分類統計（簡易圖表分析）
  - 搜尋＆篩選功能
- **Nice to Have (加分項)**：
  - 每月預算提醒
  - 自訂收支類別
  - 資料匯出報表下載
